# PayerMax Lucky Draw — 产品需求文档（PRD）

> 版本：v1.0

## 一、产品概述

PayerMax Lucky Draw 是一款面向企业年会/活动的多轮抽奖 Web 应用。通过沉浸式动画和键盘操控，主持人可以流畅地完成多个奖项轮次的现场抽奖，兼顾视觉效果与操控便捷性。

---

## 二、用户角色

| 角色 | 描述 |
|------|------|
| 主持人 | 在大屏幕前操控抽奖流程，使用键盘或鼠标推进 |
| 参与者 | 员工/嘉宾，被动参与，通过大屏幕观看结果 |

---

## 三、核心功能需求

### 3.1 抽奖轮次配置

- 系统预置 **3 个抽奖轮次**，按顺序执行：
  - **二等奖**：AirPods Pro 2，共 5 名获奖者
  - **一等奖**：iPhone 15 Pro，共 3 名获奖者
  - **特等奖**：MacBook Pro M3，共 1 名获奖者
- 每轮配置包含：轮次 ID、奖项名称、奖品名称、获奖人数、奖品图片
- 配置集中管理于 `src/config/lotteryConfig.js`，支持扩展

### 3.2 参与者数据

- 系统自动生成 **300 名模拟参与者**
- 每位参与者包含：唯一 ID、姓名（中英混合）、头像（随机）、所属部门
- 获奖者在全局范围内去重，同一人不会在多轮中重复获奖

### 3.3 抽奖流程（状态机）

系统通过 5 个状态驱动完整流程：

```
HOME → WAIT → DRAWING → RESULT → (下一轮 WAIT) → ... → HOME
```

| 状态 | 界面 | 触发操作 |
|------|------|---------|
| HOME | 欢迎页，展示 Logo + 开始按钮 | 点击按钮 / 按 Space/Enter |
| WAIT | 展示当前轮奖品信息（图片、名称、数量） | 点击"START DRAW"/ 按 Space/Enter |
| DRAWING | 钻石框滚动随机显示参与者头像和姓名 | 点击"STOP"/ 按 Space/Enter |
| RESULT | 展示本轮获奖者列表，金纸屑动画 | 点击"Next Round"/ 按 Space/Enter |
| END | 最后一轮结果确认后自动重置回 HOME | — |

### 3.4 抽奖动画

- **DRAWING 状态**：钻石旋转框内高速随机切换参与者头像 + 姓名（50ms 间隔刷新）
- **RESULT 状态**：获奖者卡片以弹簧动画逐一入场（每张延迟 0.1s）
- **金纸屑**：获奖结果揭晓时从屏幕两侧发射，持续 3 秒

### 3.5 获奖者随机选取

- 从所有未获奖的合格参与者中随机洗牌
- 按当前轮次 `matchCount` 数量截取获奖者
- 当前使用 `Array.sort(() => 0.5 - Math.random())` 简单洗牌

### 3.6 键盘控制

- **Space / Enter**：在任意状态下推进到下一步
- 监听绑定在 `window` 上，贯穿整个应用生命周期
- 状态切换时自动重新绑定，避免闭包陈旧问题

### 3.7 常驻布局

- **左侧边栏**：显示参与者总数 + 参与者名单自动滚动列表（无限循环）
- **顶部导航栏**：展示品牌名称（PayerMax）、活动名称（2025 年会盛典）、日期
- **背景效果**：深色底色 + 蓝/金渐变光晕装饰

## 四、界面规格

### 4.1 视觉风格

- **主色调**：深蓝黑色背景（`#0F172A`）
- **品牌金色**：`brand-gold`（金黄色，用于高亮、边框、按钮）
- **辅助蓝色**：`brand-blue`（用于背景光效）
- **字体**：无衬线，英文大写 tracking 设计感

### 4.2 核心 UI 组件

| 组件 | 功能 |
|------|------|
| `DiamondDisplay` | 旋转钻石框，支持 default / large 两种尺寸，可选旋转 |
| `RollingNames` | 抽奖中高速随机切换的头像+姓名展示 |
| `ScrollList` | 左侧参与者无限滚动列表 |
| `MainLayout` | 全局布局（侧边栏 + 主内容区 + 顶栏） |
| `HomeView` | 欢迎/开始页 |
| `LotteryDisplay` | 抽奖主视图（含 WAIT / DRAWING / RESULT 三态） |

## 六、已知限制 / 待优化项

| 编号 | 问题 | 优先级 |
|------|------|--------|
| L-01 | 随机算法（`0.5 - Math.random()`）统计分布不均匀 | 中 |
| L-02 | 参与者数据为 Mock 数据，无真实数据导入能力 | 高 |
| L-03 | 无音效资源文件，`soundManager` 调用无实际效果 | 低 |
| L-04 | 顶栏日期为硬编码（`2025.01.03`），非动态 | 低 |
| L-05 | 无管理后台，奖项配置需直接修改代码 | 中 |
| L-06 | 移动端适配有限，设计以全屏大屏幕为主 | 低 |
